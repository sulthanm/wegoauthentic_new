Function Requirements :

Read Ops:

- get all packages 
- get all testimonals this data should also map which package they choosed
- get testimonals based on package id
- get all gallery based on location/event choosen, 
    i) event like nature. festivals, groups, adventures
    ii) location search like Bagdogra, Thimpu
- (search bar) get packages based on search location
- get all trending destinations
- get all FAQs
- get all trips of user, based on activated time if user is added by admin.

write ops:
Users perspective : 
- form for feedback, sign in not required, (to be discuiseed). (optional field upload photo --> to be stored in gallery)   
- contact form (send mail to owner)
- customize plan requests (request should go to trips data table, status will be there)

Admin perspctive :
- add trips, update trips status.
- update gallery.
- invoice api for admin

Out of scope :
- get all trending destinations activities
- show base package when user searched location.


REST APIs:

1. GET api/v1/get-packages/ : List<Package>
2. GET api/v1/get-package/{packageId} : Package
3. POST AUTH api/v1/add-package 
	{
		packageDetails
	}
4. POST AUTH api/v1/request-trip-book : { trip_bookings } (mail or whatsapp mesg to both parties, create this user in user db)
5. GET AUTH api/v1/get-all-trips : List<Trips>
6. PUT AUTH api/v1/cancel-trip/{tripId} : Status
7. PUT AUTH api/v1/update-trip/{tripId} : { trip_bookings } 
8. GET api/v1/get-itinerary/{intenaryId} : Itinery
9. POST api/v1/add-itinerary 
10.GET api/v1/get-inclusions : List<Inclusion>
11.GET api/v1/get-exclusions : List<Exclusion>
12.POST api/v1/add-testimonal { testimonal } : if user didnt exist create user
13.GET api/v1/get-testimonal : List<Testimonal>
14.GET api/v1/get-gallery : List<Gallery>
15.POST AUTH api/v1/add-image { gallery }
16.GET api/v1/serach/{search-name} : List<Gallery> 
17.GET api/v1/get-faqs : List<FAQs>
18.POST api/v1/sign-up { user } : otp verification
19.POST api/v1/sign-in { user } : otp verification
